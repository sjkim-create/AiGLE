## 1. 화면 개요

- **화면 ID**: `AI-01`
- **화면명**: Prompt Studio
- **URL 경로**: `/ai/prompt-studio`
- **대상 사용자**: AiGLE 개발자
- **접근 권한**: 'neolab.net' 계정인 id 교사 사용자에게만 노출
- **Figma Node ID**: `미정`
- **비즈니스 목적**: 등록된 문제를 선택하고 학생 답안지를 업로드한 뒤, 시스템 프롬프트를 버전별로 관리하면서 AI 모델에 전달하여 채점 결과를 테스트하고, 품질 평가 후 아카이브에 저장할 수 있게 한다.

### 주요 사용자 흐름 (Happy Path)

1. 사용자가 Prompt Studio 화면에 진입하면 입력 파이프라인(왼쪽)과 AI 설정 사이드바(오른쪽)가 노출된다.
2. (Step 1) 등록된 과제 문항 중 하나를 선택한다. 해당 문항의 문제/모범답안/채점기준이 로드된다.
3. (Step 2) 채점할 학생의 답안 이미지 또는 JSON 필기 데이터를 업로드한다.
4. (Step 3) 평가유형(정성/정량)을 선택하고, 프롬프트 버전을 확인하거나 새 버전을 추가한다.
5. 오른쪽 사이드바에서 AI 모델(GPT/Gemini)과 파라미터를 설정한다.
6. [AI 실행] 버튼을 눌러 채점을 진행하고, 결과를 확인한다.
7. 결과 카드에서 품질 평가(평가 일치도/오류 유형)를 수행하고 [아카이브에 저장] 버튼으로 결과를 보존한다.

## 2. 화면 구성

_사용자 액션이 필요한 기능만 기술합니다. 타이틀, 라벨 등 단순 표시는 Figma에서 확인합니다._

### 등록 문항 선택 영역 (Step 1)

**전체 공통:**
- 스크롤 목록(최대 높이 160px)에서 채점할 테스트 문항 선택하여 **문항 데이터 로드**
  - 성공: 해당 문항의 문제, 모범답안, 채점기준이 하단에 표시됨. 교과에 등록된 프롬프트 버전이 Step 3에 로드됨
  - 실패: 등록된 과제가 없을 경우 목록 빈 상태
  - 조건: 필수 입력. neolab.net 계정 등록 문항만 표시. 교과순 정렬 (국어→수학→과학...). 문항 변경 시 기존 결과 및 프롬프트 상태 초기화
- 선택된 문항은 파란 배경으로 강조
- 선택 전에는 문제/모범답안/채점기준 영역 비노출

### 학생 답안 업로드 영역 (Step 2)

**전체 공통:**
- Drag & Drop Zone 또는 파일 선택을 통해 **학생 답안 업로드**
  - 성공: 이미지 미리보기로 전환
  - 실패: 지원하지 않는 파일 형식이거나 용량 초과 시 경고 문구 출력
  - 조건: 허용 파일 형식 이미지(JPG, PNG) 또는 NeoStudio 필기 데이터(JSON), 1개 파일만 업로드 가능. 필수 입력(AI 실행 버튼 활성화 조건 2)
- [✕] 버튼 클릭하여 **업로드된 파일 제거**
  - 성공: 업로드 영역이 초기 상태로 복원

### 시스템 프롬프트 영역 (Step 3 — 버전 관리)

**전체 공통:**
- [정성평가] / [정량평가] 토글 클릭하여 **평가유형 전환**
  - 성공: 선택한 평가유형 채널의 프롬프트 버전 목록으로 전환
  - 조건: `selectedAssignId` 있을 때만 표시. 기본값은 정성평가
- 버전 드롭다운에서 프롬프트 버전 선택하여 **버전 전환**
  - 성공: 해당 버전의 OCR/채점 프롬프트 내용 및 버전 설명 로드
  - 조건: `selectedAssignId` 있을 때만 표시. Live 버전은 `🔴 Live` 표시
- [+ 새 버전 추가] 버튼 클릭하여 **드래프트 모드 진입**
  - 성공: 새 버전(예: v4)이 드롭다운에 추가, "시간 미정 (미저장)" 표시. 버전 설명 입력 활성화, OCR/채점 프롬프트 영역 비어있음
  - 조건: `isDraftMode = false` 일 때만 노출
- [💾 새 버전 저장] 버튼 클릭하여 **드래프트 버전 확정 저장**
  - 성공: 실제 저장 시간 기록, 드래프트 모드 해제. DB(Live) 적용 / 삭제 버튼 노출됨
  - 조건: `isDraftMode = true` 일 때만 노출 (+ 새 버전 추가 버튼 대체)
- [✔️ DB(Live) 적용] 버튼 클릭하여 **선택 버전을 Live로 전환**
  - 성공: 해당 버전이 Live 상태로 변경, 기존 Live 해제
  - 조건: 선택 버전이 `isLive = false` + `isDraftMode = false` 일 때만 노출
- [🗑️ 삭제] 버튼 클릭하여 **선택 버전 삭제**
  - 성공: 해당 버전 삭제 후 이전 버전으로 전환
  - 조건: 선택 버전이 `isLive = false` + `isDraftMode = false` 일 때만 노출
- 📋 버전 설명 영역에서 **변경 내용 메모 입력**
  - 조건: `isDraftMode = true` 일 때만 입력 가능 (50자 이내). 그 외에는 readOnly
- OCR 프롬프트 TextArea에 **OCR 프롬프트 편집**
  - 조건: 문항 미선택 시 "문항을 선택하면 DB에 적용된 프롬프트 내용이 나옵니다." 플레이스홀더. 미등록 채널 시 "등록된 OCR 프롬프트가 없습니다." 플레이스홀더
- 채점 프롬프트 (System Prompt) TextArea에 **채점 프롬프트 편집**
  - 조건: 문항 미선택 시 "문항을 선택하면 DB에 적용된 프롬프트 내용이 나옵니다." 플레이스홀더. 미등록 채널 시 "등록된 채점 프롬프트가 없습니다." 플레이스홀더

### AI 모델 및 파라미터 설정 (우측 사이드바)

**전체 공통:**
- GPT 모델 그룹에서 모델 선택하여 **GPT 모델 지정**
  - 조건: GPT-4o mini, GPT-5.3 Instant, GPT-5.4 Thinking, GPT-5.4 Pro 중 선택
- Gemini 모델 그룹에서 모델 선택하여 **Gemini 모델 지정**
  - 조건: Gemini 3.1 Pro, Gemini 3.1 Flash-Lite 중 선택. 기본값: Gemini 3.1 Pro
- 선택된 모델: 파란 배경 pill로 강조
- `temperature` Range(0~1) 슬라이더로 **창의성/일관성 조절** (기본값: 0.1)
- `top_p` Range(0~1) 슬라이더로 **단어 선택 폭 조절** (기본값: 1.0)
- `max_tokens` Range(100~8192) 슬라이더로 **최대 토큰 수 설정** (기본값: 2048)
  - 조건: GPT / Gemini에 따라 레이블 이름 분기
- `thinking` 토글로 **사고 모드 활성/비활성** (기본값: Off)
  - 조건: GPT 모델 한정 노출
- `thinking_config` 버튼으로 **사고 설정 단계 선택**
  - 조건: Gemini 모델 한정 노출. Minimal, Low, Medium, High 4단계, 기본값: Medium

### AI 실행 버튼 영역

**전체 공통:**
- [AI 실행] 버튼 클릭하여 **채점 실행**
  - 성공: 상태를 `AI-01:AI실행중`으로 전환 및 API 호출
  - 실패: 문항 미선택 또는 답안 미업로드 시 비활성화
  - 조건: `selectedAssignId`(문항 선택)와 업로드된 답안이 모두 존재할 때 활성화. 비활성 시 "문항 선택과 답안 업로드가 완료되어야 실행할 수 있습니다." 경고 문구

### 채점 결과 카드 영역

**전체 공통:**
- `AI-01:AI실행완료` 상태 시에만 화면 하단에 동적으로 생성/노출
- OCR 변환 텍스트 및 AI 채점 결과 (Raw) 좌우 분할 표시
- 🎯 AI 채점 품질 평가 영역:
  - 평가 일치도 Select에서 값 선택하여 **평가 일치도 지정**
    - 조건: 완전 일치, 부분 일치, 불일치 중 선택
  - 오류 유형 Select에서 값 선택하여 **오류 유형 지정**
    - 조건: 해당 없음, OCR오류, Hallucination, Logic Error 등
  - 연구진 코멘트 TextArea에 **피드백 입력**
    - 조건: 플레이스홀더 "AI가 틀린 부분이나 개선이 필요한 점을 적어주세요."
- 채점 시간, 사용 토큰 정보(입력/출력/총합), 예상 비용(USD/KRW) 표시
- [↻] 버튼 클릭하여 **원화(KRW) 실시간 환율 연산 갱신**
- [📥 아카이브에 저장] 버튼 클릭하여 **분석 결과 저장**
  - 성공: 버튼 텍스트가 `✓ 아카이브 저장됨`으로 변경 및 Disabled 처리(초록색)
  - 조건: 저장 완료 후 중복 저장 방지. 저장 데이터에 promptVersionId, evalMode 등 버전/평가 정보 포함

### AI-01:문항선택_Live

- **상태 설명**: 문항 선택 완료, 해당 교과+평가유형 채널에 프롬프트 버전이 존재하며 Live 버전이 선택된 상태
- **전환 트리거**: 문항 선택 + 프롬프트 버전 존재 + Live 버전 자동 선택 → 현재 상태 진입

#### Step 1

**전체 공통:**
- 선택된 문항 파란 배경 강조
- 문제/모범답안/채점기준 데이터 표시

#### Step 3

**전체 공통:**
- 평가유형 토글 노출
- 버전 드롭다운 노출, Live 버전 선택됨 (`🔴 Live` 표시)
- `🚀 현재 Live 모델` 텍스트 표시
- DB(Live) 적용 / 삭제 버튼 비노출 (이미 Live이므로)
- [+ 새 버전 추가] 버튼 노출
- 버전 설명 비활성(readOnly)
- OCR/채점 프롬프트에 해당 버전의 실제 내용 로드

### AI-01:문항선택_미등록

- **상태 설명**: 문항 선택 완료, 해당 교과+평가유형 채널에 등록된 프롬프트가 없는 상태
- **전환 트리거**: 문항 선택 + 해당 채널 프롬프트 없음 → 현재 상태 진입

#### Step 3

**전체 공통:**
- 평가유형 토글 노출
- 버전 드롭다운 "등록 버전 없음" 표시
- DB(Live) 적용 / 삭제 버튼 비노출
- [+ 새 버전 추가] 버튼 노출
- OCR 프롬프트 플레이스홀더: "등록된 OCR 프롬프트가 없습니다."
- 채점 프롬프트 플레이스홀더: "등록된 채점 프롬프트가 없습니다."

### AI-01:드래프트

- **상태 설명**: [+ 새 버전 추가] 클릭 후, 새 버전이 미저장 상태로 편집 중인 상태
- **전환 트리거**: [+ 새 버전 추가] 클릭 → 현재 상태 진입

#### Step 3

**전체 공통:**
- 드롭다운에 `✏️ vN | 시간 미정 (미저장)` 항목 추가 및 선택됨
- [+ 새 버전 추가] → [💾 새 버전 저장] 버튼으로 교체
- DB(Live) 적용 / 삭제 버튼 비노출
- 버전 설명 입력 활성화 (플레이스홀더: "이 버전의 변경 내용을 간단히 메모하세요 (50자 이내)")
- OCR/채점 프롬프트 영역 비어있음 (직접 입력)
- 다른 버전 선택 시 → 드래프트 휘발, `isDraftMode = false`
