## 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────┐
│  Header: "Prompt Studio"  /  서브타이틀                      │
├───────────────────────────────────┬─────────────────────────┤
│  Left: 메인 파이프라인 (스크롤)    │  Right: AI 설정 사이드바 │
│                                   │                         │
│  [Step 1] 등록 문항 선택           │  🤖 AI 모델 선택         │
│  [Step 2] 학생 답안 업로드         │  ⚙️ 파라미터 설정        │
│  [Step 3] 시스템 프롬프트 (버전)   │  [AI 실행] 버튼          │
│  [Result] 분석 결과 (조건부)       │                         │
└───────────────────────────────────┴─────────────────────────┘
```

## 공통 사이드바 (우측) — 모든 상태 동일

### AI 모델 선택
- **GPT 그룹**: GPT-4o mini / GPT-5.3 Instant / GPT-5.4 Thinking / GPT-5.4 Pro
- **Gemini 그룹**: Gemini 3.1 Pro / Gemini 3.1 Flash-Lite
- 선택된 모델: 파란 배경 pill (기본값: Gemini 3.1 Pro)

### 파라미터 설정
| 파라미터 | 타입 | 기본값 |
|---------|------|--------|
| temperature | range 0~1 | 0.1 |
| top_p | range 0~1 | 1.0 |
| max_tokens | range 100~8192 | 2048 |
| thinking (GPT 선택 시) | toggle On/Off | Off |
| thinking_config (Gemini 선택 시) | 4단계 버튼: Minimal/Low/Medium/High | Medium |

### AI 실행 버튼
- **비활성화** (회색): 문항 미선택 OR 답안 미업로드
- **활성화** (진한 버튼): 문항 + 답안 모두 준비됨
- **로딩 중**: 스피너 + "AI 채점 중..." 텍스트

## SCREEN 1 — 초기 화면 (문항 미선택)

### 조건
- `selectedAssignId = ''`
- 아무것도 선택되지 않은 최초 진입 상태

### Step 1: 등록 문항 선택
```
┌─────────────────────────────────┐
│ 등록 문항 선택           [필수]  │
│                                 │
│  [중등-국어] 문학 지문 분석      │  ← 스크롤 목록 (최대 높이 160px)
│  [중등-수학] 2차 방정식 풀이     │     교과순 정렬 (국어→수학→과학...)
│  [초등-과학] 광합성 작용 원리    │     neolab.net 계정 등록 문항만 표시
│  [고등-국어] 소설 인물 분석      │
│  [고등-수학] 함수의 극값         │
└─────────────────────────────────┘
※ 선택 전: 문제/모범답안/채점기준 영역 없음
```

### Step 2: 학생 답안 업로드
```
┌─────────────────────────────────────────────┐
│ 학생 답안 업로드 (이미지 또는 JSON)  [필수 1건] │
│                                             │
│   📸                                        │
│   학생 답안 이미지 또는 JSON 필기 데이터를   │
│   업로드하세요                              │
│   1. 일반 사진(JPG, PNG 등)                 │
│   2. NeoStudio 필기 데이터(_.json)          │
└─────────────────────────────────────────────┘
```

### Step 3: 시스템 프롬프트 (버전 관리)
```
┌─────────────────────────────────────────────────────┐
│ 시스템 프롬프트 (버전 관리)         [+ 새 버전 추가] │
│                                                     │
│  📋 버전 설명  [저장된 내용이 없습니다.]              │
│                                                     │
│  OCR 프롬프트                                        │
│  ┌───────────────────────────────────────────────┐  │
│  │ 문항을 선택하면 DB에 적용된 프롬프트 내용이   │  │
│  │ 나옵니다.                                     │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  채점 프롬프트 (System Prompt)                       │
│  ┌───────────────────────────────────────────────┐  │
│  │ 문항을 선택하면 DB에 적용된 프롬프트 내용이   │  │
│  │ 나옵니다.                                     │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```
> - 평가유형 토글 없음 (문항 미선택)
> - 버전 드롭다운 없음 (문항 미선택)
> - DB(Live)적용 / 삭제 버튼 없음

### AI 실행 버튼
- 비활성화 (회색)
- 경고 문구: "⚠️ 문항 선택과 답안 업로드가 완료되어야 실행할 수 있습니다."

## SCREEN 2 — 문항 선택 후 (프롬프트 있는 경우)

### 조건
- `selectedAssignId = 'assign-002'` (예: [중등-수학] 2차 방정식 풀이)
- 해당 교과(수학)에 저장된 프롬프트 버전 존재
- `promptVersions.length >= 1`, `isLive` 버전 있음
- `isDraftMode = false`

### Step 1: 등록 문항 선택
```
┌─────────────────────────────────┐
│  [중등-국어] 문학 지문 분석      │
│  [중등-수학] 2차 방정식 풀이  ◀선택됨 (파란 배경)
│  [초등-과학] 광합성 작용 원리    │
│  ...                            │
└─────────────────────────────────┘

[문제]    x^2 - 5x + 6 = 0의 해를 구하고 풀이 과정을 쓰시오.
[모범답안] (x-2)(x-3) = 0 따라서 x = 2 또는 x = 3.
[채점기준] - 인수분해 과정의 정확성 / - 최종 해 도출 여부
```

### Step 3: 시스템 프롬프트 — Live 버전 선택 상태
```
┌──────────────────────────────────────────────────────────────┐
│ 시스템 프롬프트 (버전 관리)                                    │
│ [정성평가 ●] [정량평가]          🚀 현재 Live 모델             │
│                                                              │
│     ← 버전 드롭다운 →            [+ 새 버전 추가]             │
│  [v1 | 2026-03-25 17:20:55 🔴 Live ▼]                       │
│                                                              │
│  📋 버전 설명  [저장된 내용이 없습니다.]  ← readOnly          │
│                                                              │
│  OCR 프롬프트                                                 │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Please transcribe the handwritten Korean text...    │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
│  채점 프롬프트 (System Prompt)                                │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  ## [Role]                                           │   │
│  │  You are an expert mathematics educator...           │   │
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```
> **버튼 조건 정리**
> | 버튼 | 표시 조건 |
> |------|----------|
> | 🚀 현재 Live 모델 (텍스트) | 선택 버전이 isLive = true |
> | ✔️ DB(Live) 적용 | 선택 버전이 isLive = false + isDraftMode = false |
> | 🗑️ 삭제 | 선택 버전이 isLive = false + isDraftMode = false |
> | + 새 버전 추가 | isDraftMode = false |
> | 💾 새 버전 저장 | isDraftMode = true |
> | 평가유형 토글 | selectedAssignId 있을 때만 표시 |

## SCREEN 5 — 새 버전 추가 (Draft 상태)

### 조건
- `+ 새 버전 추가` 버튼 클릭 후
- `isDraftMode = true`
- 새 버전(예: v4)이 드롭다운에 추가됨, 시간 "시간 미정"

### Step 3: Draft 상태
```
┌──────────────────────────────────────────────────────────────┐
│ 시스템 프롬프트 (버전 관리)                                    │
│ [정성평가 ●] [정량평가]                                        │
│                                                              │
│   [✏️ v4 | 시간 미정 (미저장) ▼]    [💾 새 버전 저장]          │
│                                                              │
│   드롭다운 옵션:                                               │
│   ├─ v1 | 2026-03-25 17:20:55 🔴 Live                        │
│   ├─ v2 | 2026-03-25 18:05:22                                │
│   ├─ v3 | 2026-03-25 19:11:44                                │
│   └─ ✏️ v4 | 시간 미정 (미저장)  ← 현재 선택                  │
│                                                              │
│  📋 버전 설명  [이 버전의 변경 내용을 간단히 메모하세요 (50자 이내)] ← 입력 가능
│                                                              │
│  OCR 프롬프트                                                 │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  (비어있음 — 직접 입력)                               │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
│  채점 프롬프트 (System Prompt)                                │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  (비어있음 — 직접 입력)                               │   │
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```
> **Draft 상태 조건**
> - DB(Live) 적용 버튼 없음
> - 삭제 버튼 없음
> - `+ 새 버전 추가` → `💾 새 버전 저장` 으로 변경
> - 버전 설명 입력 가능 (활성화)
> - 다른 버전 선택 시 → draft 휘발, isDraftMode = false

## SCREEN 10 — 아카이브 저장 완료

### 조건
- `isSaved = true`
- 아카이브 저장 버튼 클릭 후

### 변경 사항
```
[✓ 아카이브 저장됨]  ← 버튼 비활성화, 초록색
```

### 저장 데이터 (Prompt 아카이브로 전달)
| 필드 | 값 |
|------|-----|
| id | TC-xxxx |
| title | [과제명] 테스트 |
| category | 교과 (국어/수학/과학) |
| model | 선택한 AI 모델 |
| evalMode | 선택한 평가유형 (정성평가/정량평가) |
| promptVersionId | 현재 활성 버전 ID (예: v2) |
| promptVersionDatetime | 해당 버전 저장 시간 |
| promptSubject | 교과명 |
| latency / tokens / costUsd | AI 실행 결과 |
| matchStatus | 평가 일치도 (수동 선택) |
| errorType | 오류 유형 (수동 선택) |
| teacherFeedback | 연구진 코멘트 |
| ocrText | OCR 변환 결과 |
| gradingResult | AI 채점 결과 Raw |

### 데이터 흐름
```
Prompt Studio
    │
    │  [📥 아카이브에 저장] 클릭
    │  → onSaveArchive(archiveItem) 호출
    │
    ▼
Prompt 아카이브 메뉴
    - 저장된 테스트 목록에 추가
    - 버전 정보 (promptVersionId, datetime) 포함
    - 평가 일치도 / 오류 유형 분석 가능
```
